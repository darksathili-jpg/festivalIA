Exposition Interactive et Participative

Six stands immersifs, conçus et animés par les élèves de Première NSI, explorent la créativité de l'intelligence artificielle à travers l'image, le texte et la musique — et interrogent le rôle du jugement humain.

À propos de l'exposition

Dans le sillage des fêtes galantes d'Antoine Watteau, où la frontière entre la nature et l'artifice, entre le jeu et le sérieux, restait délicieusement floue — cette exposition invite chacun à interroger la frontière entre création humaine et génération artificielle. Les élèves de Première NSI endossent ici le rôle des experts. Vous êtes le jury.

Expérimenter

Chaque stand propose une interaction directe : tester, comparer, voter.

Comprendre

Les élèves expliquent comment l'IA fonctionne — sans jargon inutile.

Décider

Droits d'auteur, biais, authenticité : forgez votre propre opinion.

La Galerie

Les Six Stands de l'Exposition

Sélectionnez un stand pour découvrir le déroulé pédagogique complet.

A

Stand A · Image

Watteau contre l'IA

L'IA génère

Des tableaux dans le style rococo d'Antoine Watteau — figures élégantes, jardins, lumière dorée.

Le visiteur

Compare avec les vraies oeuvres du maître et tente de distinguer l'original de l'imitation.

Notions NSI mobilisées

API REST Requêtes HTTP JSON Données images

Déroulé du stand (15 min)

  • L'élève animateur présente huit images sur l'écran : quatre vraies oeuvres de Watteau et quatre générées par l'IA via un script Python utilisant l'API OpenAI Images, hébergée côté serveur.
  • Les visiteurs votent sur une feuille ou via un formulaire numérique ; les résultats sont affichés en direct.
  • Révélation : les réponses sont dévoilées et le taux de réussite affiché.
  • Discussion guidée : qu'est-ce qui trahit l'IA ? Qu'est-ce que le « style » ?

Questions de réflexion

Si l'IA reproduit parfaitement le style de Watteau, qui est l'auteur de l'image ?
Peut-on déposer un copyright sur un tableau généré par IA ?

Outils techniques

Python 3 API OpenAI Images (relais serveur) Requests Interface HTML
B

Stand B · Texte

Le Poème Impossible

L'IA génère

Des poèmes sur Valenciennes, le lycée Watteau, la jeunesse — en alexandrins, haïkus, vers libres.

Le visiteur

Évalue chaque poème selon trois critères : émotion ressentie, authenticité perçue, originalité.

Notions NSI mobilisées

API LLM Ingénierie du prompt Chaînes de caractères Collecte de données

Déroulé du stand (15 min)

  • Le visiteur soumet un thème ou un mot-clé (par exemple : « le beffroi », « un dimanche matin », son prénom).
  • L'élève animateur génère en direct un poème via l'API Claude ou GPT, en montrant le prompt utilisé. L'appel passe par un relais serveur local — aucune clé API n'est exposée dans le navigateur.
  • Le visiteur note le poème sur cinq étoiles selon trois critères : émotion, authenticité, originalité.
  • Un poème écrit par un être humain sur le même thème est ensuite présenté pour comparaison.

Questions de réflexion

Un poème peut-il toucher si son auteur n'a jamais rien ressenti ?
Le prompt que j'écris fait-il de moi le co-auteur du poème ?

Outils techniques

Python 3 API Claude / GPT (relais serveur) Interface HTML CSV de résultats
C

Stand C · Musique

Qui a Composé Cela ?

L'IA génère

Des mélodies courtes au clavecin, dans le style baroque du XVIIIe siècle — l'époque de Watteau.

Le visiteur

Un test d'écoute à l'aveugle : parmi six extraits, lesquels sont de Couperin ou de Rameau ? Lesquels sont générés ?

Notions NSI mobilisées

Données audio API musicale Fichiers binaires Randomisation

Déroulé du stand (15 min)

  • Six extraits de trente secondes sont joués depuis un Raspberry Pi branché à un haut-parleur.
  • Trois extraits sont de vrais morceaux baroques (Couperin, Rameau, Bach) ; trois sont générés par Suno AI ou MusicGen.
  • Le visiteur inscrit ses paris sur un bulletin. Les résultats sont révélés à la fin.
  • Discussion : la « qualité » musicale suffit-elle à définir l'oeuvre d'art ?

Questions de réflexion

Si une mélodie générée provoque de l'émotion, est-elle de l'art ?
Les compositeurs baroques suivaient des règles strictes — en quoi l'IA diffère-t-elle ?

Outils techniques

Raspberry Pi Suno AI / MusicGen Pygame Fichiers MP3
D

Stand D · Portrait

Mon Portrait en Dix Styles

L'IA génère

Le portrait du visiteur transposé dans dix styles artistiques différents : rococo, manga, cubisme, art pixel…

Le visiteur

Choisit son portrait préféré, puis discute de ce que chaque style révèle — ou efface — de son identité.

Notions NSI mobilisées

Images numériques API génération Raspberry Pi Vie privée / RGPD
Conformité RGPD — Données biométriques Ce stand collecte des photographies de visage, qualifiées de données biométriques au sens du RGPD. Le protocole suivant est obligatoire : (1) le consentement explicite et libre du visiteur est recueilli avant toute prise de vue ; (2) les personnes mineures doivent disposer de l'accord d'un responsable légal présent ; (3) la photographie est supprimée définitivement à l'issue de la session, sous les yeux du visiteur ; (4) aucune image n'est conservée, transmise ou réutilisée. Ces étapes font partie intégrante du déroulé pédagogique et illustrent concrètement les enjeux du RGPD.

Déroulé du stand (20 min)

  • L'élève présente le cadre RGPD et recueille le consentement écrit du visiteur avant toute action.
  • Le visiteur accepte (ou refuse librement) d'être photographié sur fond neutre avec le Raspberry Pi équipé d'une caméra.
  • Un script Python transmet la photographie à l'API de génération via un relais serveur local, avec dix prompts de style différents.
  • Les dix portraits s'affichent sur l'écran en quelques secondes. Le visiteur choisit son préféré.
  • L'élève supprime la photographie originale en direct, à la vue du visiteur, et explique pourquoi cette étape est légalement requise.

Questions de réflexion

Lorsque l'IA transforme mon visage, est-ce encore moi sur l'image ?
Que deviennent les données biométriques utilisées pour entraîner ces modèles ?

Outils techniques

Raspberry Pi 5 Caméra Pi API OpenAI Images (relais serveur) Python / Pillow Écran HDMI
E

Stand E · Fiabilité

L'IA Hallucine

L'IA génère

Des « informations » sur Valenciennes, le lycée Watteau, des personnages historiques — dont certaines sont délibérément fausses.

Le visiteur

Doit repérer les erreurs, les inventions, les confabulations — et comprendre pourquoi l'IA les produit.

Notions NSI mobilisées

Modèles de langage Probabilités Données d'entraînement Vérification des sources

Note pédagogique — Fiches simulées

Avertissement éditorial Toutes les fiches présentant des informations erronées portent la mention visible « DOCUMENT PÉDAGOGIQUE SIMULÉ — À NE PAS DIFFUSER » afin d'éviter toute confusion hors du contexte du stand. Cette précaution fait partie intégrante du protocole.

Déroulé du stand (15 min)

  • Cinq fiches d'information sont présentées sur des thèmes locaux ou historiques. Deux contiennent des erreurs fabriquées par l'IA, clairement marquées « simulation pédagogique ».
  • Le visiteur identifie les fiches suspectes, sans aide extérieure dans un premier temps.
  • L'élève explique le mécanisme de l'hallucination : l'IA prédit des tokens vraisemblables, non des faits vérifiés.
  • Démonstration en direct : la même question est posée à l'IA et sa réponse — correcte ou inventée — est analysée.

Questions de réflexion

Peut-on faire confiance à un outil qui ne « sait » pas qu'il se trompe ?
Comment vérifier rapidement une information générée par IA ?

Outils techniques

Claude / GPT HTML statique Fiches imprimées Vidéoprojecteur
F

Stand F · Écriture

Écris avec Moi

L'IA propose

Un début de récit — une scène dans les jardins de Versailles, un matin de juin 1715.

Le visiteur

Continue l'histoire à tour de rôle avec l'IA, en alternant les paragraphes. Le résultat s'affiche en direct.

Notions NSI mobilisées

Conversation multi-tour Contexte API Interface web Co-création

Déroulé du stand (20 min)

  • L'interface affiche le début du récit sur l'écran partagé. Le visiteur saisit son paragraphe de suite.
  • L'IA répond automatiquement via un relais serveur local. L'alternance se poursuit jusqu'à six tours (trois humains, trois IA).
  • Le récit complet est imprimé sur une carte souvenir ou transmis par courriel si le visiteur le souhaite.
  • L'élève montre le prompt système qui oriente l'IA pour demeurer dans le registre de l'époque.

Questions de réflexion

Qui est l'auteur de l'histoire finale — moi, l'IA, ou les deux ?
L'IA a-t-elle modifié ma façon d'écrire en s'y adaptant ?

Outils techniques

API Claude (relais serveur) Interface HTML/JS Historique de messages Impression thermique

Compétences NSI de Première Mobilisées

Chaque stand est un projet concret ancré dans le programme officiel.

Représentation des données

  • Encodage des images (pixels, RVB, formats)
  • Représentation des données audio (fréquence, échantillonnage)
  • Données textuelles et encodage Unicode
  • Formats d'échange JSON et CSV

Algorithmique et Programmation

  • Fonctions et modules Python
  • Traitement de chaînes de caractères
  • Gestion de fichiers et de données
  • Appels asynchrones et gestion d'erreurs

Web et Interactions

  • Requêtes HTTP (GET / POST)
  • Utilisation d'API REST avec clés d'accès
  • HTML, CSS et JavaScript pour les interfaces
  • Collecte et affichage de données en temps réel

Société et Numérique

  • Propriété intellectuelle et droits d'auteur
  • Protection des données personnelles (RGPD)
  • Biais algorithmiques et représentativité
  • Fiabilité des sources et vérification de l'information
